# Phase 6 — Public Beta + App Store launch

**Status**: ⏳ Planned
**Estimated days**: Weeks 5–6
**Depends on**: Phase 5 done (App Store submission in review or approved)
**Goal**: App Store listing goes live, run the build-in-public launch sequence, and gather signal on whether the reciprocity-lock mechanic actually works for indie builders. The app is now publicly installable by anyone.

Maps to VISION.md §8 Phase 6 and §10 (success metrics + kill criteria).

## Subtasks

1. App Store listing live (the app is publicly installable). TestFlight Public Link as fallback while review pending.
2. X build-in-public launch sequence — daily drops with screenshots and shareable profile URLs (`commit.app/keanu`), peer mentions (Melwin, Patrick, Tom)
3. Indie Hackers + r/SideProject + Hacker News Show HN posts (with App Store + profile URL links)
4. Discord server for beta feedback
5. **Listening mode — no new features** during this phase. Bugs only. (VISION §8 is explicit about this.)
6. Cohort analytics dashboard tracking: Day-7 retention, average drops per active user per day, % of drops that include the photo (proves the mechanic isn't being bypassed)
7. Apply success-metric thresholds from VISION §10.2: 50+ users, 30%+ Day-7 retention, 2+ drops/user/day, 80%+ drops with photo

## End state

50–100 beta users with both qualitative feedback (Discord) and quantitative signal (cohort retention numbers), all installed via the public App Store listing. Decision point: continue to V1 public release, pivot, or hit kill criteria from VISION §10.4 (Day-30 < 15%, drops/user/day < 1.0, or founders themselves stop using).
